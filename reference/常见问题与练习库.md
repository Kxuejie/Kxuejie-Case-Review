# Case 复盘 · 常见问题检测清单 + 练习方法库

> **来源**：真实辅导点评（toy market entry / airline / pharma 三场录音）。这些是这位教练反复看到的高频问题和真实给出的练习方法。
> **用法**：
> - **Phase 8（重大问题清单）**：把"一、常见问题检测清单"当**检测清单**逐项扫描面试者表现，命中即报，用"老师会怎么说"里的语气点出来。
> - **Phase 9（后续练习建议）**：按命中的问题，从"二、练习方法库"匹配对应训练。
> - **重点提升方向**：用"三、重点提升方向判定"挑出最该突破的方向 + Top 2-3 能马上改善的点。

---

## 一、常见问题检测清单（按 Case 阶段）

> 逐项扫描。命中的写进报告，没命中的不用提。每条给了"老师会怎么说"的真实语气，命中时照这个调子点。

### A. Clarify 阶段

- **A1 · 漏问目标**（一票否决）
  - 触发：clarify 里没问 objective / 具体数字 / 时间限制。
  - 老师会怎么说："目标都没搞清楚就开始拆框架，后面所有分析都是悬空的。"

- **A2 · 漏问 scope**
  - 触发：题目只给了现有业务，面试者没确认"是只基于现有业务做提升，还是可以考虑新业务/组合调整"。
  - 老师会怎么说："你到底是基于现有的这些（航线/业务）做利润率提升，还是也可以考虑新的、组合的调整？这个 scope 你没问，导致你后面整个产品组合的视角就丢了。"

- **A3 · 漏问 geography / 产销区**
  - 触发：跨地域/跨国业务，没问产区销区覆盖、是不是 global。
  - 老师会怎么说："你的产区跟销区是什么情况？这个每道题都该问一下——你如果一开始问了是 global，后面 brainstorm 自然会带出地理扩展这个点。"

- **A4 · Clarify 问太多 / 太散 / 超范围**
  - 触发：问题超过 3 个，或天马行空超出 clarify 范畴。
  - 老师会怎么说："Clarify 不是让你把整道题问完，3 个以内，问完赶紧进框架。"

- **A5 · 问问题不解释为什么问**
  - 触发：直接抛问题，没说"我问这个是因为……"。
  - 老师会怎么说："你问之前先说为什么问、先说你的假设，再问。"

### B. 框架阶段

- **B1 · 不熟悉行业，框架套乱 / 套了过大框架**
  - 触发：换到不熟悉的行业（航空、医药、to-B 等），框架明显比熟悉的消费品乱；或对小问题套了过大的框架。
  - 老师会怎么说："你之前消费品做得好，是因为拆 revenue、拆 cost、举措都很类似。换个行业收入方式不一样，分析方向就不一样，你就没那么熟练了——框架在新行业要会变形。"

- **B2 · 第二层没说子点数量**
  - 触发：进入一个板块时没说"这板块下我看 N 个因素"，第一点直接展开很长。
  - 老师会怎么说："你介绍框架不光要说有几个主板块，每个板块下面有几个子点也要先说出来，比如'行业分析我看三个因素'，再展开。"

- **B3 · 机会/评估类把目的设太死，不讨论优劣势**（Opportunity Assessment 高频）
  - 触发：OA 类题把目的设太绝对，只 identify 一个点（如"能不能提升 compliance"），不讨论这个 option 的优劣势 / 正反两面。
  - 老师会怎么说："你这个目的设得太强、太绝对了。Opportunity assessment 不是 profitability turnaround，不能 narrow 到一个点——优势是啥、劣势是啥，正反两面都要说。compliance 只是它的优势之一。"

- **B4 · 缺"新营收 / 其他好处"这个可加板块**
  - 触发：增长/利润/机会类题，没有"除了直接目的，还有没有其他好处/新营收来源（synergy、长期战略价值）"。
  - 老师会怎么说："任何利润提升、营收提升，永远可以加一个——除了这个直接的增长/目的，还有没有别的好处？synergy、长期战略价值。题目不会明说，但你提一嘴就是 think outside the box。"

### C. 推进 / Brainstorm 阶段

- **C0 · brainstorm 普遍不足 / 推进太被动**（高频核心问题，最该先看）
  - 触发：每进一个板块不先自己 brainstorm 给假设就直接问面试官要数据；分析完一个板块不做阶段性 finding 小结就往下走；全程等面试官抛问题，case 推得太快草草结束；说完框架不问面试官 OK 就闷头往下走。
  - 注：**敢下结论 ≠ 强观点**。学员常给出"不收购""用净资产法"这种结论，但背后没有 brainstorm 出来的一串依据支撑——结论没有佐证就是空的，顶多算中等观点。这一条最容易被误判成"有观点、很强"。
  - 老师会怎么说："你这个 case 推得太快、太被动了，基本是面试官抛一个你接一个。每个板块你应该先自己 brainstorm 一串假设——size 大概多大、growth 怎么样、利润率什么水平——再去跟面试官要数据验证。而且每分析完一块要有个小结论（finding），不然整道题就是一堆零散的问答，没有你自己的判断在里面。你结论是敢下的，但凭什么下？背后没有 brainstorm 支撑，结论就是空的。"

- **C1 · 公司自身能力 brainstorm 不足**
  - 触发：行业那块 brainstorm 很好，但分析公司自身能力时太薄，没从 4P（Product/Price/Place/Promotion）或渠道协同/管理协同展开。
  - 老师会怎么说："行业那块 brainstorm 很好，但公司自身这块你 brainstorm 少了。从 4P 去想，渠道现在是经销还是直销、有没有协同，这些都是能力评估点。"

- **C2 · 风险 brainstorm 不足**
  - 触发：分析最后风险点太少。
  - 老师会怎么说："风险和下一步可以再多一点。"

- **C3 · 目标贯穿性缺失**
  - 触发：题目给了明确目标（如营收提升 10%），但全程**完全没有回扣**这个目标。
  - 注：只要学员中途回扣了目标方向（哪怕没量化到"占多少份额"）就算合格，**不用苛求量化路径**——回扣了目标就不错了。
  - 老师会怎么说："你算完得有一句回到目标——这离 10% 还差多少。完全不提目标，整道题就飘着。"

### D. 计算阶段

- **D1 · 四步流程不全**
  - 触发：拿到数直接算，跳过"确认问题 / 对齐信息 / 列逻辑"。
  - 老师会怎么说："拿到数先 rephrase question 跟面试官确认，再 align 一下所有 information（包括你自己估的 margin），再把计算逻辑列出来，然后一步一步、能拆细就拆细，带着面试官 work through。"

- **D2 · 不 round up 简化心算**
  - 触发：用了很难的数硬算，没取整。
  - 老师会怎么说："你估的时候别给自己一个很难的数，可以跟面试官说能不能 round up 一下。"

- **D3 · 算错**
  - 触发：最终结果算错。
  - 老师会怎么说："小偏差（2%-3%）无所谓，前面流程做对了面试官会给你找借口；但偏差太大（10%）就影响整体了——所以前面流程起码要对。"

### E. 总结 / 风险 / 下一步阶段

- **E1 · 风险和下一步完全跳过**
  - 触发：为省时间，risk & next step 一句没说。
  - 老师会怎么说："你这次风险和下一步完全没说。哪怕笼统讲一下也行，别让它显得像丢了一块。"

- **E2 · 风险说太细，没统一成 ROI 概念**
  - 触发：每个措施的风险都展开讲一长串。
  - 老师会怎么说："每个手段都有相应的 ROI，风险你统一说就行，不用每个都讲特别细。"

- **E3 · 缺固定话术（3 大板块先报）**
  - 触发：风险下一步没有结构，零散地说。
  - 老师会怎么说（正确范式）："风险和下一步我从 market、financial、implementation 三个方向说，这里暂不展开，最后给 recommendation 时统一补全。"——先报三个板块，就不会显得空。

### F. 行业认知（根因层）

- **F1 · 行业认知薄是很多问题的根因**
  - 触发：很多表层问题（框架乱、漏问、推进慢）其实根因是不懂这个行业的底层经济逻辑。
  - 举例：
    - 航空：不理解"上座率/载客率是命门、成本端（折旧/机场费）很难降、每条航线是一个 SKU"，所以收入涨但利润率跌看不懂。
    - 医药：不理解"冷链能扩大地理覆盖（可以卖到更远的国家）"，所以漏掉地理这个好处。
  - 老师会怎么说："你不是没问 scope 的问题，根上是你对这个行业不够理解——它的利润率为什么收入涨了反而跌，这个底层逻辑你没吃透。"

## 二、练习方法库（按问题簇匹配）

> 每条：练什么 / 怎么练 / 练多久。命中哪簇问题就开哪个处方。

### 处方 1 · 不同/复杂行业专项（对应 B1、F1）
- 练什么：把框架迁移到不熟悉行业的"变形"能力。
- 怎么练：刻意挑**不熟悉、收入结构不一样的行业**（to-B、航空、医药、重资产这种），别只刷消费品。每做一道，先写下"这个行业的钱是怎么赚的、命门是什么"，再套框架。
- 多久：2-3 周，每周 2-3 道不同行业，覆盖至少 4 个陌生行业。

### 处方 2 · Clarify 固定模板（对应 A1、A2、A3）
- 练什么：每道题开口前固定问全：**目标（含数字/时间）+ scope（现有 vs 新业务）+ 产销区/geography**。
- 怎么练：拿 5-8 道不同类型 case prompt，只做 clarify 部分，限定 3 个问题，每个都写出"我问这个是因为……"。
- 多久：5-8 道，集中一周练完。

### 处方 3 · 计算四步流程 + 心算（对应 D1、D2、D3）
- 练什么：确认问题 → 对齐信息 → 列逻辑 → 逐步算；以及 round up 简化。
- 怎么练：专练 break-even / ROI / payback period 三类计算。每道强制走四步，开口前先 round up；心算先熟 10%、再熟 1%、再叠加。
- 多久：每天 1 道计算题，连续 5 天。

### 处方 4 · 机会/评估类第二层改造（对应 B3、B4）
- 练什么：OA / 机会类固定加"其他好处（synergy、长期战略价值）"板块，且正反两面都说。
- 怎么练：找 2-3 道 Opportunity Assessment，每道在 attractiveness 里强制分成"主要目的 + 其他好处"，且每个 option 列优势 ≥2 / 劣势 ≥2。
- 多久：2-3 道。

### 处方 5 · 风险下一步固定话术（对应 E1、E2、E3）
- 练什么：风险/下一步先报 3 大板块（market / financial / implementation），统一说不展开。
- 怎么练：拿任意 3 道做过的 case，只重练结尾——开口就"风险和下一步我从这三个方向说……"，每个板块各放 1-2 个风险，最后统一补全。
- 多久：3 道。

### 处方 6 · 观点结构三段式（对应 C1、C2、C3 及表达类问题）
- 练什么：每拿到一条信息，强制"事实 → 这意味着什么（因果）→ 所以应该（行动）"，并回扣目标。
- 怎么练：找 2 道 Profitability/Growth，每次拿到数字先说一句"这意味着……，所以我们离目标还差……"再继续。
- 多久：每天 1 道，连续 5 天。

### 处方 7 · Brainstorm 专项 + 主动推进（对应 C0，最该练）
- 练什么：在向面试官要数据之前，先自己 brainstorm 出一串假设和依据；每个板块分析完做一个阶段性 finding 小结；让结论"有佐证"而不是光下结论。
- 怎么练：拿一道 case，**只看 prompt + clarify 拿到的信息，先不看后面的数据**——自己把框架列出来，每个板块下强制写出"我 brainstorm 的假设是什么、为什么这么猜"，写在纸上；然后再对照 case 实际数据，检查自己漏了哪些点、假设差多远。再加 1-2 次完整 mock，刻意练"每块先 brainstorm 给假设 → 要数据验证 → 给 finding 小结 → 推进核心问题"的节奏。
- 多久：先用 2-3 道做"只 brainstorm 不看数据"的纸面练习，再加 1-2 次完整 mock。

## 三、重点提升方向判定

> Phase 9 输出"重点提升方向"时用。**先按学员水平定大方向**，再给几个具体举动 + Top 2-3 能马上改善的点。**不要排"第 X 周练什么"的周计划。**

### 第一步：先判断学员基础水平

看基础维度：框架介绍方式、框架质量、观点强度、计算流程、Clarify。

- **基础薄弱**（这些基础维度多数 ≤5 分）→ 重点方向 = **回到原则里的基础动作，先把流程和框架练扎实**：
  - 背框架骨架（各题型核心逻辑要背熟，如 Market Entry"先该不该进再怎么进"）
  - 框架介绍方式（先报层级和子点数量）
  - 观点形成（事实 → 因果 → 行动）
  - 计算流程（四步 + round up）
  - 对应处方 2、3、6 + 背框架骨架。
  - ⚠️ **不要给基础薄弱的学员开"补行业认知"**——他还没到那一步。

- **基础扎实**（基础维度多数 ≥6-7 分、流程和框架都稳）→ 重点方向才是**进阶**：
  - 行业认知（陌生行业的钱怎么赚、命门是什么）
  - 框架在陌生行业的变形能力
  - 对应处方 1。

> 一句话：**行业认知是"基础够好了再上"的进阶项；基础没打牢之前，重点永远是框架和流程。**

### 第二步：在大方向下挑几个具体举动

按命中的问题簇，从"二、练习方法库"挑对应处方，给 3-5 个具体举动。一句话讲清"练什么 + 怎么练 + 大概练多少"，但**不排周计划**。

特殊优先项（命中就提到重点里）：
- **A 类一票否决（A1 漏目标）命中** → 不管基础好坏，先修 clarify（处方 2）。
- **C0 命中（brainstorm 不足 / 推进太被动）** → 不管基础好坏，brainstorm 专项（处方 7）都是重点。这是核心基本功，不是"行业认知"那种进阶项——别因为学员"敢下结论"就放过他。
- **某个维度拿了 1-3 分** → 那个维度就是当下最该补的洞。

### Top 2-3 "能马上改善"的点怎么挑
挑那些**靠固定动作/话术就能立刻见效**、不需要长期积累的：
- Clarify 固定三问（处方 2）——下一道立刻能用。
- 风险下一步固定话术（处方 5）——下一道立刻能用。
- 计算先 round up（处方 3 的一部分）——下一道立刻能用。
- 机会类固定加"其他好处"板块（处方 4）——下一道立刻能用。

> 区分：**重点突破方向**是慢功夫；**马上改善的点**是快招（固定动作/话术），下一道 case 就能用上。两个都要给。
